# Alphabet Language — Conference Talk Proposals

## Talk 1: "Alphabet: A Multilingual Programming Language"

**Abstract:**
What if you could code in Amharic? Alphabet is a new programming language that lets you write code in English, Amharic, Spanish, French, or German. This talk covers the design decisions, challenges, and impact of building a multilingual language.

**Key Points:**
- Why multilingual programming matters
- Keyword mapping across 5 languages
- Voice input for accessibility
- Teaching programming to non-English speakers

**Duration:** 30 minutes + Q&A
**Audience:** Language designers, educators, accessibility advocates

---

## Talk 2: "Voice Input for Programming: Speaking Code"

**Abstract:**
Alphabet introduces voice-to-text input for programming. Users can speak code in any of 5 supported languages, and the NL-to-Code converter translates natural language to executable code. This talk covers the technical implementation and user experience.

**Key Points:**
- Speech-to-text integration (Vosk + Whisper)
- NL-to-Code conversion pipeline
- Voice-assisted hybrid mode
- Accessibility benefits

**Duration:** 20 minutes + Q&A
**Audience:** HCI researchers, accessibility engineers, voice interface designers

---

## Talk 3: "Teaching Programming with Alphabet"

**Abstract:**
Alphabet is designed for education. With simple syntax, multilingual support, and interactive tutorials, it makes programming accessible to beginners. This talk covers pedagogical design decisions and classroom experience.

**Key Points:**
- Simple syntax (19 keywords)
- Full keywords for beginners (if==i, else==e)
- Interactive REPL with tutorials
- 30 built-in examples
- Unit testing framework

**Duration:** 45 minutes + hands-on workshop
**Audience:** CS educators, curriculum designers, bootcamp instructors

---

## Talk 4: "Building a Language with C++17 Bytecode VM"

**Abstract:**
Alphabet compiles to bytecode and runs on a custom VM built in C++17. This talk covers the architecture: lexer, parser, compiler, VM, and tooling. Learn how to build your own language.

**Key Points:**
- Lexer/parser/compiler architecture
- Bytecode VM with 45 opcodes
- Garbage collection
- LSP server integration
- Embedding API

**Duration:** 45 minutes + Q&A
**Audience:** Systems programmers, language enthusiasts, compiler engineers
